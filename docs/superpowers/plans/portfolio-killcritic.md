# Biggest Weakness: Lack of Proof

Your site talks about capabilities.

It doesn't show evidence.

For example:

> "Dedicated to build robust and scalable backend systems and APIs." ([Gio Majadas | Personal Portfolio][1])

A recruiter doesn't care about claims.

They care about proof.

Show:

- API throughput
- Database scale
- Number of users
- Performance improvements
- Deployment architecture
- Screenshots
- GitHub repositories

Instead of:

> I build scalable APIs.

Say:

> Built REST APIs serving 50,000+ requests/month using ASP.NET Core and PostgreSQL.

Even if the numbers are smaller, real metrics beat generic statements.
